# Spec : Internationalisation (i18n) — NotifCar Landing

**Date :** 2026-06-27
**Scope :** Ajout du support FR / EN / ES sur l'ensemble du site vitrine NotifCar

---

## 1. Objectif

Permettre aux visiteurs de consulter le site en français, anglais et espagnol via un sélecteur de langue dans le header. Toutes les pages et composants sont traduits, y compris les pages légales (CGU, Politique de confidentialité).

---

## 2. Librairies

| Package | Rôle |
|---|---|
| `i18next` | Moteur i18n |
| `react-i18next` | Hooks React (`useTranslation`) |
| `i18next-browser-languagedetector` | Détecte la langue du navigateur au premier chargement |

---

## 3. Structure des fichiers

```
src/
  i18n.ts                  ← configuration i18next
  locales/
    fr.json                ← textes français (source de vérité)
    en.json                ← traductions anglaises
    es.json                ← traductions espagnoles
  components/
    ui/
      LanguageSwitcher.tsx ← sélecteur FR🇫🇷 / EN🇬🇧 / ES🇪🇸
docs/
  superpowers/
    specs/
      2026-06-27-i18n-design.md  ← ce fichier
```

---

## 4. Configuration i18next (`src/i18n.ts`)

- Langue par défaut : `fr`
- Fallback : `fr`
- Détection : `localStorage` (clé `notifcar-lang`) → langue navigateur → fallback
- Namespaces : un seul namespace `translation` (site vitrine simple)
- Ressources chargées statiquement (import JSON) — pas de lazy loading nécessaire à ce stade

---

## 5. Clés de traduction

Les clés sont organisées par section/composant. Exemple de structure dans `fr.json` :

```json
{
  "header": { "download": "Télécharger l'app", "nav": { ... } },
  "hero": { "title": "...", "subtitle": "...", "cta": "..." },
  "howItWorks": { "badge": "...", "title": "...", "steps": [...] },
  "features": { ... },
  "faq": { "title": "...", "items": [...] },
  "pricing": { ... },
  "contact": { ... },
  "footer": { ... },
  "legal": { "privacy": { ... }, "cgu": { ... } }
}
```

Les données actuellement dans `src/data/faq.ts` sont déplacées dans les fichiers JSON (`faq.items`).

---

## 6. Sélecteur de langue (`LanguageSwitcher`)

- Intégré dans `Header.tsx` entre la nav desktop et le bouton CTA
- Affiche les codes langue : `FR` / `EN` / `ES` (avec drapeaux emoji)
- Persistance via `localStorage` (`notifcar-lang`)
- Sur mobile : visible dans le menu burger, en bas des liens de nav

---

## 7. URLs

Les URLs restent inchangées (`/tarifs`, `/contact`, `/confidentialite`, `/cgu`). Pas de préfixe `/fr/`, `/en/` pour ne pas casser les liens existants et simplifier la navigation.

## 9. Traductions générées

Des traductions EN et ES sont générées directement dans les fichiers JSON — elles peuvent être affinées manuellement ensuite. Le fichier FR sert de source de vérité.

## 10. Ce qui ne change pas

- Structure des routes React Router
- Style et layout des composants
- Logique métier (AuthContext, EmailService, etc.)
- Assets (images, logos)
